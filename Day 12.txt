## 1ï¸âƒ£ `useEffect` Hook

### ğŸ” What is it?

The `useEffect` hook allows you to perform **side effects** in function components.
Examples of side effects:

* Fetching data
* Updating the DOM
* Using timers
* Subscribing/unsubscribing to events

---

### ğŸ§  Syntax

```js
useEffect(() => {
  // side effect logic
  return () => {
    // optional cleanup
  };
}, [dependencies]);
```

---

### ğŸ§ª Examples

#### ğŸ“Œ Run once on component mount (like `componentDidMount`)

```js
useEffect(() => {
  console.log('Component mounted');
}, []);
```

#### ğŸ” Run when specific state changes

```js
useEffect(() => {
  console.log('Count updated:', count);
}, [count]);
```

#### ğŸ§¹ Cleanup example

```js
useEffect(() => {
  const interval = setInterval(() => {
    console.log('Tick');
  }, 1000);

  return () => clearInterval(interval); // cleanup
}, []);
```

#### ğŸŒ Fetching data

```js
useEffect(() => {
  fetch("https://jsonplaceholder.typicode.com/users")
    .then(res => res.json())
    .then(data => setUsers(data));
}, []);
```

## 2ï¸âƒ£ Rendering Lists

### ğŸ” Why?

To display **multiple items** dynamically from an array.

---

### âœ… Basic Example

```js

  <ul>
    {fruits.map((fruit, index) => (
      <li key={index}>{fruit}</li>
    ))}
  </ul>
);
```

### âš ï¸ Key Prop

---

### ğŸ§ª More Complex Example

## 3ï¸âƒ£ Handling Forms in React

### ğŸ” Controlled vs Uncontrolled

* **Controlled**: React manages input state
* **Uncontrolled**: DOM manages state (with `ref`)

---

### âœ… Controlled Form Example

```js
const [name, setName] = useState('');

return (
  <form onSubmit={(e) => { e.preventDefault(); alert(name); }}>
    <input
      type="text"
      value={name}
      onChange={(e) => setName(e.target.value)}
    />
    <button type="submit">Submit</button>
  </form>
);
```

---

### ğŸ§ª Form with Multiple Inputs

```js
const [formData, setFormData] = useState({ email: '', password: '' });

const handleChange = (e) => {
  setFormData({
    ...formData,
    [e.target.name]: e.target.value
  });
};
```

```jsx
<form>
  <input name="email" onChange={handleChange} />
  <input name="password" onChange={handleChange} />
</form>
```

---

### ğŸ§ª Select, Radio, and Checkbox

```jsx

  <option value="female">Female</option>

```


## ğŸ§ª Uncontrolled Form Example

Unlike controlled forms (where state tracks inputs), in an **uncontrolled form**, the data is accessed directly from the **DOM** using a `ref`.

---

### âœ… Example Code:

```jsx
import React, { useRef } from 'react';

function UncontrolledForm() {
  const nameRef = useRef(null);
  const emailRef = useRef(null);

  const handleSubmit = (e) => {
    e.preventDefault();
    const name = nameRef.current.value;
    const email = emailRef.current.value;

    alert(`Name: ${name}\nEmail: ${email}`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <div>
        <label>Name:</label>
        <input type="text" ref={nameRef} />
      </div>
      <div>
        <label>Email:</label>
        <input type="email" ref={emailRef} />
      </div>
      <button type="submit">Submit</button>
    </form>
  );
}

export default UncontrolledForm;
```

---

### ğŸ§  Explanation:

* `useRef()` is used to create a reference to the input DOM nodes.
* `ref={nameRef}` attaches the reference to the input.
* On form submission, `nameRef.current.value` directly accesses the DOM value.
* This avoids React state management â€” hence **â€œuncontrolled.â€**
